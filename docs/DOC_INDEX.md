- Update `MVP_STATUS.md` when prototype status changes.
- Update `MVP_0_2_PLAN.md` if MVP 0.2 scope changes.
- Update `MISSION_DECK.md` when mission content changes.
- Update `UX_SPEC.md` when route or interaction behaviour changes.
- Update `ARCHITECTURE.md` when services, state, or validation changes.
- Update `API_CONTRACT.md` when route payloads change.
- Update `RESPONSIBLE_AI_DESIGN.md` when model behaviour or public claims change.
- Update `TEST_PLAN.md` when readiness gates or failure cases change.
- Update `RUNBOOK.md` when launch, reset, fallback, or shutdown changes.
- Keep `AGENTS.md` short and actionable.
